Learn about Windows 10 IoT Core

Windows 10 IoT Core is a version of Windows 10 that is optimized for smaller devices with or without a display, and that runs on the Raspberry Pi 2, Arrow DragonBoard 410c & MinnowBoard MAX. Windows 10 IoT Core has a rich, extensible, Universal Windows Platform (UWP) API for building great solutions.Â 


Runs on a variety of hardware

You can run Windows 10 IoT Core on a wide variety of hardware, supporting both x6 and ARM chipsets. You can develop on Raspberry Pi 2, MinnowBoard MAX and Qualcomm DragonBoard 410c. You can also control an Arduino with a Windows app.

Windows capabilities and development model

Develop Universal Windows Apps as you would for Windows on desktop or mobile and run them on small devices. Windows 10 IoT Core provides the same capabilities you love about Windows: security, updates, tools, apps and manageability, and uses the same development tools for Windows 10.

Tools and languages of your choice

Seamless integration with Powerful tools like Visual Studio. Support for C#, C++, Python, .NET and Node.js. Consolidated with Threshold OS core, tools and processes and one standardized Windows 20 IoT Core image for all products.

Simple cloud connectivity

Seamless integration with Azure cloud using connect-the-dots framework.


Hardware Requirements

The requirements for Windows 10 IoT Core depend on whether it is run in a headed or headless mode. Headed devices have a video display and use Windows video subsystem to address it. Headless devices have no display.

Process

400 MHz or faster

x86 requires PAE, NX and SSE2 support)

Memory

Headless

256 MB RAM (128 MB free to OS) / 2 GB Storage

Headed

512 MB RAM (256 MB free to OS) / 2 GB Storage
